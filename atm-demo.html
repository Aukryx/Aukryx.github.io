About The Project

The ATM Management System is a comprehensive banking solution built in C that simulates the functionality of an automated teller machine. It provides secure user authentication, account management, transaction processing, and data persistence.

C Programming File Handling Data Structures User Authentication Financial Systems Terminal UI

Key Features

Secure Authentication

Multi-level user authentication with password masking and validation to protect financial data and user privacy.

Account Management

Create, update, and delete bank accounts with different types and configurations. View detailed account information at any time.

Transaction Processing

Secure handling of deposits and withdrawals with balance verification and transaction validation to prevent errors.

Ownership Transfer

Securely transfer account ownership between users while maintaining account history and integrity.

Data Persistence

All account information and transaction history is saved between sessions using a robust file-based storage system.

Multi-User Support

Register new users and manage multiple accounts per user with different access levels and permissions.

Technical Implementation

This project demonstrates several key programming concepts and techniques:

Modular Architecture

The system is built with a modular design separating authentication, account management, and transaction processing into distinct components (as seen in the project's Makefile structure).

File-Based Database

Custom implementation of a file-based storage system that maintains data integrity between sessions while providing fast access to account information.

Input Validation

Comprehensive validation of all user inputs to prevent errors, ensure data integrity, and enhance security.

Error Handling

Robust error detection and handling for situations like insufficient funds, invalid credentials, or file access issues.
